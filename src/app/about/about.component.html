A homelab, on purpose.

databaes.net is a personal homelab run by Tomas Ferreira. It started as one mini-PC in a corner and grew into a small cluster that hosts the same kind of services a mid-sized team would outsource: streaming, file sync, identity, observability, and a control plane to run the rest.

This page is the short version — the longer story lives in the wiki and the git log.

Why a homelab

Three reasons, in order: data sovereignty (photos, notes, and calendars shouldn't depend on someone else's ToS), learning in public (the fastest way to understand a system is to run one), and a playground for production patterns (SSO, GitOps, blue-green deploys, AI-assisted remediation — the stuff that wins in real operations teams).

The homelab is small enough that one person can understand every layer, and opinionated enough that every layer actually makes a decision. No YAML shoveled onto a cluster without a reason.

The stack, briefly

hosts Proxmox VE · bare-metal mini-PC cluster
orchestration k3s · Rancher · Traefik · Cloudflare Tunnels
identity Authentik SSO (OIDC + forwardAuth) · Vault (secrets)
apps Go microservices · Angular + SvelteKit frontends · PostgreSQL · NATS
observability Prometheus · Loki · Tempo · Grafana · OpenTelemetry
gitops GitHub Actions · self-hosted runners · blue-green worker slots
